## 二、云端→设备（设备需订阅）

| 主题 | 说明 | Payload |
|------|------|---------|
| `retail/{dev}/cmd/unlock` | 远程开门 | `{"action":"UNLOCK","duration":30}` |
| `retail/{dev}/cmd/lock` | 远程锁门 | `{"action":"LOCK"}` |
| `retail/{dev}/cmd/sync` | 主动拍照同步 | `{"action":"SYNC"}` |
| `retail/{dev}/cmd/reboot` | 重启设备 | `{"action":"REBOOT"}` |
| `retail/{dev}/msg/err` | 异常通知(亮红灯) | `{"type":"SUSPICIOUS_OBJECTS","level":"WARN"}` |

> `msg/err` 是消息通知频道，设备收到后亮红灯即可。type 区分异常类型，level 为 `WARN`/`CRIT`
> OTA 升级通过 HTTP 下载固件，不走 MQTT

## 三、设备→云端（设备需上报）

### 3.1 识别结果（开关门/同步）

| 项目 | 值 |
|------|-----|
| 主题 | `retail/{dev}/event/detect` |

**开门识别：**
```json
{
  "batch_id": "uuid-xxx",
  "door_action": "OPEN",
  "detections": [
    {"label": "nongfu", "score": 0.95, "x1": 100, "y1": 200, "x2": 300, "y2": 400},
    {"label": "soda",   "score": 0.88, "x1": 150, "y1": 250, "x2": 350, "y2": 450}
  ]
}
```

**关门识别：**
```json
{
  "batch_id": "uuid-xxx",
  "door_action": "CLOSE",
  "detections": [
    {"label": "soda", "score": 0.88, "x1": 150, "y1": 250, "x2": 350, "y2": 450}
  ]
}
```

**同步拍照识别：**
```json
{
  "batch_id": "sync-xxx",
  "door_action": "SYNC",
  "detections": [...]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| batch_id | string | **必填**。同一购买事件的唯一ID（UUID），OPEN和CLOSE必须一致 |
| door_action | string | `OPEN` / `CLOSE` / `SYNC` |
| detections | array | 检测结果数组 |
| detections[].label | string | `nongfu` / `runtian` / `soda` / `yogurt` |
| detections[].score | number | 置信度 0-1 |
| detections[].x1,y1,x2,y2 | number | 边界框坐标（像素） |

### 3.2 OTA 升级进度

| 项目 | 值 |
|------|-----|
| 主题 | `retail/{dev}/event/ota_progress` |

```json
{
  "type": "model",
  "version": "v1.1",
  "status": "downloading",
  "progress": 45,
  "message": ""
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | `firmware` 固件 / `model` 模型 |
| version | string | 版本号 |
| status | string | `downloading` / `installing` / `success` / `failed` |
| progress | number | 进度百分比 0-100 |
| message | string | 失败时填写错误信息 |

## 四、HTTP 接口

### 4.1 拍照上传（备用，云端推理时使用）

| 项目 | 值 |
|------|-----|
| URL | `POST https://<server-ip>/api/snapshot/upload` |
| 鉴权 | Token `<snapshot-token-D01>`（免 Basic Auth） |
| 字段 | token, deviceId(`D01-F1`), floor(1), doorAction(`OPEN`/`CLOSE`/`SYNC`), file |

### 4.2 背景图上传

| 项目 | 值 |
|------|-----|
| URL | `POST https://<server-ip>/api/snapshot/background/upload` |
| 鉴权 | Token `<snapshot-token-D01>` |
| 字段 | token, deviceId(`D01-F1`), file（空柜照片） |

> 上传空柜照片作为异常检测基线。开门关门时自动比对，区域差异>35%标记异常。

### 4.3 OTA 固件/模型下载

| 项目 | 值 |
|------|-----|
| URL | `GET https://<server-ip>/api/firmware/{id}/download` |
| 鉴权 | Basic Auth `admin:<nginx-basic-auth-password>` |
| 返回 | 文件二进制流 |

## 五、业务逻辑

```
一次购买事件 = 开门 → 关门（batch_id 串联）

① 开门 → ESP32拍照识别 → MQTT上报 OPEN + batch_id → 服务端缓存计数
② 顾客拿取饮料
③ 关门 → ESP32拍照识别 → MQTT上报 CLOSE + batch_id
④ 服务端: diff = OPEN - CLOSE → 生成订单 → 更新库存
```
