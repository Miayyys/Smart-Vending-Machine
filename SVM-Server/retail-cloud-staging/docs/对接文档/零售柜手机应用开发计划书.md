# 校园无人零售柜 — 手机应用开发计划书

> **项目背景**：云平台（Spring Boot + ONNX AI + MySQL + MQTT）已上线运行，Web 管理端功能完备。现需开发手机 App 对标 Web 端全部功能，支持离线使用。

---

## 一、技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| 开发框架 | React Native 0.76+ | 跨平台 iOS/Android，一套代码 |
| 语言 | TypeScript | 类型安全，降低运行时错误 |
| 导航 | @react-navigation v7 | RN 官方推荐，Tab + Stack |
| 状态管理 | Zustand | 轻量无模板，比 Redux 简洁 5 倍 |
| 离线存储 | WatermelonDB (SQLite) | 高性能，支持懒加载和增量同步 |
| HTTP | Axios | 拦截器方便处理 Basic Auth |
| MQTT | react-native-mqtt | 基于 WebSocket，连接 EMQX |
| 相机 | react-native-vision-camera | 高性能，支持拍照和扫码 |
| 图片缓存 | react-native-fast-image | 自动缓存缩略图 |
| 图表 | react-native-chart-kit | 仪表盘统计卡片可视化 |

## 四、各页面功能详述

### 4.1 仪表盘

| 功能 | 说明 |
|------|------|
| 统计卡片 ×4 | 在线设备数、商品种类、低库存告警数、未处理异常数 |
| 最近订单 ×5 | ID、设备名、金额、支付状态、创建时间 |
| 最近告警 ×6 | 类型（补货/异常）、设备、详情、时间 |
| 自动刷新 | 每 30 秒轮询 + MQTT 实时推送更新统计数 |
| 下拉刷新 | 手动拉取最新数据 |

### 4.2 库存管理

| 功能 | 说明 |
|------|------|
| 柜选择器 | 下拉选择 1号柜 / 2号柜 |
| 柜级聚合 | 各楼层 F1/F2/F3 库存汇总显示 |
| 楼层展开 | 点击商品行展开/收起各层明细 |
| 缺货高亮 | 数量=0 红色、≤阈值 橙色、正常绿色 |
| 改单价 | 点击价格 → 弹窗修改 → 调 API 保存 |
| 库存校准 | 点击行 → 弹窗输入新数量 → 均分到各层 |

### 4.3 订单管理

| 功能 | 说明 |
|------|------|
| 倒序列表 | 最新订单在最上面 |
| 柜筛选 | 下拉选择全部/1号柜/2号柜 |
| 展开明细 | 点击「展开」显示商品、单价、数量、小计 |
| 编辑用户 | 点击用户名 → 弹窗修改 |
| 确认付款 | 待支付订单显示按钮 → 点击后写结算时间 |
| 状态标识 | 已支付(绿)/待支付(黄)/失败(红) |

### 4.4 告警管理

**补货告警（子 Tab 1）**

| 功能 | 说明 |
|------|------|
| 状态筛选 | 待处理 / 已确认 / 全部 |
| 列表展示 | 柜子、商品名、柜级库存、阈值、时间 |
| 一键确认 | PENDING 状态可点「确认补货」→ 写确认时间 |
| 0 库存高亮 | 库存为 0 时红色加粗 |

**异常告警（子 Tab 2）**

| 功能 | 说明 |
|------|------|
| 筛选 | 仅显示未处理 / 全部 |
| 列表展示 | 设备、类型、级别(INFO/WARN/CRIT)、详情、时间 |

### 4.5 拍照记录

| 功能 | 说明 |
|------|------|
| 筛选条件 | 柜子、楼层、动作（开门/关门） |
| 缩略图列表 | 80×60 缩略图 + AI 检测结果文字 |
| 大图查看 | 点击缩略图 → 全屏查看 |
| 本机拍照 | 调用手机相机 → 选柜/楼层/动作 → 上传 |
| 检测状态 | 缩略图旁显示「待检测/处理中/检测结果」 |

## 五、离线策略

### 数据分层

```
┌─────────────────────────────────────┐
│ 内存 (Zustand)     UI 实时状态      │ ← 最快
├─────────────────────────────────────┤
│ SQLite (WatermelonDB)  完整数据快照 │ ← 离线可用
├─────────────────────────────────────┤
│ 文件系统 (RNFS)    图片缓存          │ ← 大文件
└─────────────────────────────────────┘
```

### 同步机制

```
📶 在线模式：
  API 拉取 → 更新内存 → 同步写入 SQLite
  30s 自动轮询 → 比较时间戳 → 增量更新

📴 离线模式：
  纯读 SQLite → 显示缓存数据
  顶部「离线模式」横幅提示
  写操作按钮灰色禁用

🔁 恢复在线：
  先推离线排队的操作 → 再拉最新数据 → 刷新 UI
```

### 网络检测
- 使用 `@react-native-community/netinfo` 监听
- 断网 → Toast 提示 + 3 秒后自动隐藏
- 恢复 → 静默同步，不打扰用户

## 六、后台对接

### 无需新增 API

现有 Spring Boot 后台 API 完全满足移动端需求，无需任何改动：

| 模块 | API | 方法 |
|------|-----|------|
| 商品 | `/api/products` | GET |
| 商品 | `/api/products/{id}` | GET / PUT |
| 库存 | `/api/devices/{id}/stock` | GET |
| 库存校准 | `/api/stock/reconcile` | POST |
| 订单 | `/api/orders` | GET / POST |
| 订单更新 | `/api/orders/{id}/update` | POST |
| 订单明细 | `/api/orders/{id}/items` | GET |
| 补货告警 | `/api/alerts/restock` | GET |
| 确认补货 | `/api/alerts/restock/{id}/ack` | POST |
| 异常告警 | `/api/alerts/anomaly` | GET |
| 拍照上传 | `/api/snapshot/upload` | POST (Multipart + Token) |
| 拍照列表 | `/api/snapshot/search` | GET |
| 拍照图片 | `/api/snapshot/{id}/file` | GET |

### 鉴权方式
- Web 功能：HTTP Basic Auth（Axios 自动携带）
- 拍照上传：设备专用 Token（免 Basic Auth）

### MQTT 实时推送
- 服务器：EMQX `wss://主机/mqtt`
- 订阅主题：`retail/+/event/stock_detected`、`retail/+/event/anomaly`、`retail/+/event/snapshot`
- 收到推送 → 直接更新 Zustand Store → UI 自动刷新
