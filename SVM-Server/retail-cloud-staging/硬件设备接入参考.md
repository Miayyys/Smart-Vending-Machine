# 硬件设备接入参考手册

> 适用版本：2026-06-29 · 线上 `<server-ip>` · 每个楼层 MCU 独立设备 ID
> 后端源码字段名逐字核对，与本文件不一致以本文件为准。

---

## 1. 连接参数

### 1.1 MQTT Broker

```
地址:   tcp://<server-ip>:1883
账号:   retail_device
密码:   <mqtt-device-password>
QoS:    1（发布和订阅都用 1）
KeepAlive: 30s（推荐）
CleanSession: true
```

### 1.2 HTTP API（抓拍上传 / OTA 下载）

```
Base URL:  https://<server-ip>
认证:      Basic Auth  admin / <nginx-basic-auth-password>
          （仅抓拍上传免 Basic Auth，用设备 token 替代，见 §5）
证书:      自签，设备端需跳过 TLS 校验或信任自签证书
```

## 3. 上传方向（设备 → 云，MQTT Publish）

Topic 格式：`retail/{device}/event/{sub}` 或 `retail/{device}/telemetry/{sub}`

### 3.1 取货事件（扣库存）

```
Topic:   retail/{device}/event/take
QoS:     1
Payload: JSON
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `pid` | number | **是** | 商品 ID（注意字段名是 `pid`，不是 `productId`） |
| `act` | string | 否 | 动作，默认 `"TAKE"`（可取 `"TAKE"` 或 `"PUTBACK"`） |
| `qty` | number | 否 | 数量，默认 `1` |
| `w` | number | 否 | 重量(克)，可空 |

**示例：**
```json
{"pid": 1, "act": "TAKE", "qty": 1, "w": 550}
{"pid": 3, "act": "TAKE", "qty": 2, "w": 140}
{"pid": 1, "act": "PUTBACK", "qty": 1, "w": 550}
```

> 后端动作：扣 `qty` 库存 → 写 `take_event` 表 → 若库存跌破 `threshold` → 自动建 `restock_alert`

### 3.2 结算事件（建订单）

```
Topic:   retail/{device}/event/settle
QoS:     1
Payload: JSON
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `uid` | string | 否 | 用户标识（校园卡号等），可空 |
| `amount` | number | 否 | 订单总金额，默认 `0`（省则后端从商品表查价自动算） |
| `status` | string | 否 | 支付状态，默认 `"PAID"`（取值 `UNPAID`/`PAID`/`FAILED`） |

**示例：**
```json
{"uid": "2024001", "amount": 9.50, "status": "PAID"}
{"uid": "2024002", "status": "PAID"}
```

> 后端动作：`order_info` 建单 + 自动查商品价补明细

### 3.3 异常告警

```
Topic:   retail/{device}/event/anomaly
QoS:     1
Payload: JSON
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 否 | 异常类型，默认 `"UNKNOWN"`（建议填具体：`DOOR_FORCED`/`WEIGHT_MISMATCH`/`TAKE_EXCEED` 等） |
| `level` | string | 否 | 级别，默认 `"WARN"`（`INFO`/`WARN`/`CRIT`） |
| (任意) | any | 否 | 其他字段会整包存入 `payload` 字段 |

**示例：**
```json
{"type": "DOOR_FORCED", "level": "WARN"}
{"type": "WEIGHT_MISMATCH", "level": "CRIT", "expected": 550, "actual": 320}
{"type": "TEMP_HIGH", "level": "INFO", "temp": 42.5}
```

> 后端动作：入 `anomaly_alert` 表

### 3.4 称重遥测（仅记日志，不落业务库）

```
Topic:   retail/{device}/telemetry/weight
QoS:     1
Payload: JSON
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `w` | number | **是** | 重量(克) |

**示例：**
```json
{"w": 550}
```

> 后端动作：仅 DEBUG 日志输出，不写数据库。用于调试称重传感器。

## 4. 下发方向（云 → 设备，MQTT Subscribe）

设备必须订阅自己楼层的下发指令：

```
订阅 Topic:  retail/{device}/cmd/#
QoS:         1
```

收到后解析 JSON 的 `action` 字段，执行对应动作。

### 4.1 远程开门

```
Topic:    retail/{device}/cmd/unlock
Payload:  {"action": "UNLOCK", "duration": 30}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | string | 固定 `"UNLOCK"` |
| `duration` | number | 开门持续秒数 |

### 4.2 远程锁门

```
Topic:    retail/{device}/cmd/lock
Payload:  {"action": "LOCK"}
```

### 4.3 重启设备

```
Topic:    retail/{device}/cmd/reboot
Payload:  {"action": "REBOOT"}
```

### 4.4 同步库存

```
Topic:    retail/{device}/cmd/sync
Payload:  {"action": "SYNC_STOCK"}
```

> 设备收到后应上报当前每格库存快照（可通过一连串 take/putback 校准，或直接调 REST reconcile）

### 4.5 更新价格

```
Topic:    retail/{device}/cmd/price
Payload:  {"action": "UPDATE_PRICE", "productId": 1, "price": 2.50}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | string | 固定 `"UPDATE_PRICE"` |
| `productId` | number | 商品 ID |
| `price` | number | 新单价(元) |

### 4.6 补货完成

```
Topic:    retail/{device}/cmd/restock
Payload:  {"action": "RESTOCK_DONE", "items": [{"productId": 1, "qty": 10}, {"productId": 2, "qty": 8}]}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | string | 固定 `"RESTOCK_DONE"` |
| `items` | array | 补货清单 |
| `items[].productId` | number | 商品 ID |
| `items[].qty` | number | 补货数量 |

> 设备收到后可同步本地库存显示；云端库存可通过 `POST /api/stock/reconcile` 手动对齐

### 4.7 OTA 固件升级

```
Topic:    retail/{device}/cmd/ota
Payload:  {"action": "OTA_UPDATE", "version": "v1.2.3", "url": "https://<server-ip>/api/firmware/1/download", "md5": "a1b2c3d4...", "size": 123456}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | string | 固定 `"OTA_UPDATE"` |
| `version` | string | 固件版本号 |
| `url` | string | 固件下载地址（HTTPS） |
| `md5` | string | MD5 校验值（32位十六进制小写） |
| `size` | number | 文件字节数 |

**设备收到后应：**
1. HTTP GET `url` 下载固件（**需带 Basic Auth 头** `admin:<nginx-basic-auth-password>`，跳过 TLS 校验）
2. 计算下载文件的 MD5，与 `md5` 字段比对
3. 校验通过 → 刷写固件 → 重启
4. 刷写成功/失败后可上报 `retail/{device}/event/anomaly` type=`OTA_SUCCESS`/`OTA_FAIL` 回执（当前后端不强制，二期待加）

## 5. HTTP 楼层抓拍图片上传

**不走 MQTT**，用 HTTP multipart POST。**免整站 Basic Auth**，用设备专属 token。

### 5.1 上传接口

```
POST  https://<server-ip>/api/snapshot/upload
Content-Type: multipart/form-data
跳过 TLS 校验（自签证书）
```

| 表单字段 | 类型 | 必填 | 说明 |
|----------|------|------|------|
| `token` | string | **是** | 设备上传 token（见下表） |
| `deviceId` | string | **是** | 设备 ID，必须与 token 绑定的设备一致 |
| `floor` | int | **是** | 楼层号，≥1 |
| `doorAction` | string | **是** | `OPEN` 或 `CLOSE`（大小写不敏感） |
| `file` | file | **是** | 图片文件（jpg/png/gif/webp） |

### 5.2 各楼层 token

| device ID | token |
|-----------|-------|
| D01-F1 | `<snapshot-token-D01-F1>` |
| D01-F2 | `<snapshot-token-D01-F2>` |
| D01-F3 | `<snapshot-token-D01-F3>` |
| D02-F1 | `<snapshot-token-D02-F1>` |
| D02-F2 | `<snapshot-token-D02-F2>` |
| D02-F3 | `<snapshot-token-D02-F3>` |
| D01（父） | `<snapshot-token-D01>` |
| D02（父） | `<snapshot-token-D02>` |

### 5.3 curl 示例

```bash
curl -sk \
  -F "token=<snapshot-token-D01-F1>" \
  -F "deviceId=D01-F1" \
  -F "floor=1" \
  -F "doorAction=OPEN" \
  -F "file=@/path/to/photo.jpg" \
  https://<server-ip>/api/snapshot/upload
```

### 5.4 成功响应 (200)

```json
{
  "id": 123,
  "url": "/api/snapshot/123/file",
  "fileName": "photo.jpg",
  "fileSize": 45678,
  "md5": "a1b2c3d4e5f6...",
  "floor": 1,
  "doorAction": "OPEN",
  "createTime": "2026-06-29T12:00:00"
}
```

### 5.5 错误响应

| 情况 | HTTP | 消息 |
|------|------|------|
| 缺 token | 401 | `{"error": "missing token"}` |
| 错 token | 401 | `{"error": "invalid token"}` |
| token 与 deviceId 不匹配 | 403 | `{"error": "token not for this device"}` |
| floor < 1 或缺 | 400 | `{"error": "invalid floor"}` |
| doorAction 不是 OPEN/CLOSE | 400 | `{"error": "doorAction must be OPEN or CLOSE"}` |
| 缺 file | 400 | `{"error": "empty file"}` |

> 上传成功后，后端自动通过 MQTT `retail/{device}/event/snapshot` 通知前端即时刷新。
> 每设备自动保留最近 200 张，超出删最旧。

## 6. OTA 固件下载（设备端 HTTP GET）

收到 `cmd/ota` 下发指令后，设备用 HTTP GET 下载固件：

```
GET   https://<server-ip>/api/firmware/{id}/download
Header:  Authorization: Basic <base64(admin:<nginx-basic-auth-password>)>
         → 即 Authorization: Basic <base64(admin:密码)>
跳过 TLS 校验（自签证书）
```

响应为二进制固件流（`Content-Type: application/octet-stream`），下载后计算 MD5 与指令中的 `md5` 比对。

## 8. 鉴权总览

```
接入层:
  MQTT 1883 ─── retail_device / <mqtt-device-password>  ─→ 发布 retail/+/event/#
                 (设备账号)                      ─→ 订阅 retail/{本机}/cmd/#

  HTTPS 443 ─── nginx Basic Auth admin / <nginx-basic-auth-password> ─→ 保护全部 /api/*
              └── POST /api/snapshot/upload 豁免 Basic Auth
                  改用设备 token 鉴权（device_token 表）

  EMQX Dashboard 18083 ─── 仅绑 127.0.0.1，不对公网开放
```

## 9. 典型设备端代码骨架（伪代码）

```c
// ===== 1. 初始化 =====
wifi_connect("SSID", "PASSWORD");
mqtt_connect("tcp://<server-ip>:1883", "D01-F1",
             "retail_device", "<mqtt-device-password>");

// ===== 2. 订阅下发指令 =====
mqtt_subscribe("retail/D01-F1/cmd/#", QOS=1);
// 回调: 收到 JSON → 解析 action →
//   UNLOCK→开门, LOCK→锁门, REBOOT→重启,
//   OTA_UPDATE→HTTP下载+MD5校验+刷写

// ===== 3. 开门 → 拍照 → 上传 =====
camera_capture("floor1_open.jpg");
http_post_multipart("https://<server-ip>/api/snapshot/upload", {
    token:      "<snapshot-token-D01-F1>",
    deviceId:   "D01-F1",
    floor:      1,
    doorAction: "OPEN",
    file:       "floor1_open.jpg"
}, skip_tls=true);

// ===== 4. 检测取货 → 上报 =====
mqtt_publish("retail/D01-F1/event/take",
    "{\"pid\":1,\"act\":\"TAKE\",\"qty\":1,\"w\":550}", QOS=1);

// ===== 5. 关门 → 拍照 → 结算 =====
mqtt_publish("retail/D01-F1/event/settle",
    "{\"uid\":\"card_0001\",\"amount\":9.50,\"status\":\"PAID\"}", QOS=1);

camera_capture("floor1_close.jpg");
http_post_multipart(".../api/snapshot/upload", {
    token: "<snapshot-token-D01-F1>",
    deviceId: "D01-F1",
    floor: 1,
    doorAction: "CLOSE",
    file: "floor1_close.jpg"
}, skip_tls=true);

// ===== 6. 异常检测 → 上报 =====
mqtt_publish("retail/D01-F1/event/anomaly",
    "{\"type\":\"DOOR_FORCED\",\"level\":\"WARN\"}", QOS=1);

// ===== 7. 心跳/称重（可选） =====
// 每 30s:
mqtt_publish("retail/D01-F1/telemetry/weight", "{\"w\":550}", QOS=1);
```

## 10. 快速验证命令（服务器上测试用）

```bash
# SSH 到服务器
ssh user@<server-ip>

# 模拟 D01-F1 上报取货（公网路径）
mosquitto_pub -h <server-ip> -p 1883 \
  -u retail_device -P '<mqtt-device-password>' \
  -t 'retail/D01-F1/event/take' \
  -m '{"pid":1,"qty":1,"w":550}'

# 模拟 D01-F1 订阅接收下发（另开终端）
mosquitto_sub -h <server-ip> -p 1883 \
  -u retail_device -P '<mqtt-device-password>' \
  -t 'retail/D01-F1/cmd/#' -v

# 模拟下发开门指令
mosquitto_pub -h <server-ip> -p 1883 \
  -u retail_ui -P '<mqtt-ui-password>' \
  -t 'retail/D01-F1/cmd/unlock' \
  -m '{"action":"UNLOCK","duration":30}'
```
