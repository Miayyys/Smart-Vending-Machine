# 部署配置清单

克隆仓库后，**必须**修改以下配置项才能正常部署。按端分类，每项标注文件路径和行内关键词方便搜索。

---

## 一、硬件端 (P4HarfWare)

### `P4HarfWare/platform1/main/config.h`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| WiFi SSID | `your-wifi-ssid` | 设备连接的 WiFi 名称 |
| WiFi 密码 | `your-wifi-password` | WiFi 密码 |
| 服务器 IP | `your-server-ip` | 云服务器公网 IP 或域名（出现 2 次: SERVER_HOST 和 MQTT_BROKER_URI） |
| MQTT 密码 | `your-mqtt-password` | 需与服务器 EMQX 设备账号密码一致 |
| 设备上传 Token | `your-snapshot-token` | 需与服务器 `device_token` 表中该设备的 token 一致 |
| 设备 ID | `D01-F1` | 按实际柜号修改（格式 `{柜号}-F{楼层}`） |

> MQTT 用户名保留 `retail_device` 不变（除非服务器端 EMQX 改了账号名）。

### `P4HarfWare/platform1/main/bus/state_machine.cpp`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| OTA 下载密码 | `your-ota-password` | 需与服务器 nginx Basic Auth 密码一致 |

### `P4HarfWare/platform1/CMakeLists.txt`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| ESP-DL 路径 | `EXTRA_COMPONENT_DIRS` | 修改为本地 ESP-DL 安装路径 |

### `P4HarfWare/platform1/main/network/generate_cert.sh`

**固件编译前必须执行**，生成 TLS 证书用于 HTTPS 通信：

```bash
cd P4HarfWare/platform1/main/network
bash generate_cert.sh <服务器IP>
```

生成的 `server_cert.pem` 会被 CMakeLists.txt 嵌入固件。服务器 IP 变更后需重新生成。

## 二、服务器端 (SVM-Server)

部署第一步：复制环境变量模板并填入实际密码。

```bash
cd SVM-Server/retail-cloud-staging
cp .env.example .env
# 编辑 .env 修改所有 <...> 占位符
chmod 600 .env
```

### `SVM-Server/retail-cloud-staging/.env`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| MySQL root 密码 | `MYSQL_ROOT_PASSWORD` | 数据库 root 密码 |
| MySQL app 密码 | `MYSQL_PASSWORD` | 应用数据库密码（两处同名变量） |
| EMQX Dashboard 密码 | `EMQX_DASHBOARD_PASSWORD` | EMQX 管理后台登录密码 |
| EMQX UI 密码 | `EMQX_UI_PASSWORD` | 管理后台/后端连接 MQTT 的密码 |
| EMQX 设备密码 | `EMQX_DEV_PASSWORD` | 硬件设备连接 MQTT 的密码 |
| 快照保留数 | `SNAPSHOT_KEEP` | 每设备保留拍照张数，默认 200 |

### `SVM-Server/retail-cloud-staging/.htpasswd`

Nginx Basic Auth 凭证文件，用于保护管理后台。生成命令：

```bash
htpasswd -c .htpasswd admin   # 交互式输入密码
```

### `SVM-Server/retail-cloud-staging/emqx/init-users.sh`

EMQX 账号初始化脚本，需将脚本中的用户名/密码替换为 `.env` 中的对应值。

### `SVM-Server/retail-cloud-staging/frontend/js/mqtt.js`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| MQTT 密码 | `your-mqtt-password` | 需与 `.env` 中 `EMQX_UI_PASSWORD` 一致 |

### `SVM-Server/retail-cloud-staging/docker-compose.yml`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| 端口绑定 | `127.0.0.1:` | 需要对公网开放的服务去掉 `127.0.0.1:` 前缀 |

## 三、手机端 (app/sx)

### `app/sx/app.js`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| 服务器地址 | `your-server-ip` | 云服务器 IP 或域名（出现 3 处） |
| Basic Auth | `your-base64-credentials` | `admin:密码` 的 Base64 编码 |

生成 Base64 凭证：

```bash
echo -n "admin:你的密码" | base64
```

### `app/sx/utils/cert.js`

| 配置项 | 关键词 | 说明 |
|--------|--------|------|
| HTTPS 证书 | `your-certificate-base64` | 服务器自签证书的 Base64 编码 |

生成命令：

```bash
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes -subj "/CN=你的服务器IP"
base64 -w0 cert.pem
```

## 五、配置对应关系

各端密码需保持一致：

```
┌─────────────────────────────────────────────────────────┐
│ 密码用途                  │ 硬件端          │ 服务器端        │ 手机端         │
├───────────────────────────┼─────────────────┼─────────────────┼────────────────┤
│ MQTT 设备连接密码         │ config.h        │ .env (EMQX_DEV) │ —              │
│ MQTT 管理连接密码         │ —               │ .env (EMQX_UI)  │ —              │
│                           │                 │ mqtt.js         │                │
│ HTTPS Basic Auth 密码     │ state_machine   │ .htpasswd       │ app.js         │
│                           │ .cpp (OTA下载)  │                 │ (adminAuth)    │
│ 设备上传 Token            │ config.h        │ device_token 表 │ —              │
│                           │ (SNAP_TOKEN)    │ (MySQL)         │                │
└─────────────────────────────────────────────────────────┘
```

## 六、检查清单

部署前逐项确认：

- [ ] 硬件端 `config.h` 中 WiFi、服务器 IP、MQTT 密码、Token 已修改
- [ ] 硬件端 `state_machine.cpp` 中 OTA 密码已修改
- [ ] 硬件端 `CMakeLists.txt` 中 ESP-DL 路径正确
- [ ] 硬件端 `generate_cert.sh` 已执行，`server_cert.pem` 已生成
- [ ] 服务器端已执行 `cp .env.example .env` 并修改所有密码为实际值
- [ ] 服务器 `.env` 中所有密码已修改且不为默认值
- [ ] 服务器 `.htpasswd` 已生成
- [ ] 服务器 `emqx/init-users.sh` 中账号密码与 `.env` 一致
- [ ] 服务器 `mqtt.js` 中 MQTT 密码与 `.env` 一致
- [ ] 手机端 `app.js` 中服务器地址和 Basic Auth 已修改
- [ ] 手机端 `cert.js` 中 HTTPS 证书已替换
- [ ] 所有端密码对照"配置对应关系"表确认一致
