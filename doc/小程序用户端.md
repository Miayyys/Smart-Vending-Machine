# 校园智能柜 — 用户端小程序介绍

## 1. 概述

微信小程序，项目名"校园智能柜"，为用户提供浏览商品、扫码开门、查看订单、支付结算的一站式购物体验。与硬件端（ESP32-P4 智能柜）和服务器端配合，实现完整的无人零售闭环。

- **技术栈**：微信小程序原生框架，Skyline 渲染引擎，Glass-Easel 组件框架
- **服务器**：通过 HTTP API 通信，Basic Auth 认证
- **设备**：绑定设备 D01-F1，开门请求通过服务器 MQTT 转发至硬件端

## 3. 核心业务流程

### 3.1 用户登录

首次启动检查本地存储的 `userInfo`。若不存在，自动跳转登录页。登录后 `userId`（UUID，匿名生成）、`userInfo`、`token` 持久化到本地存储。退出登录后清除 `userInfo` 和 `token`，保留 `userId` 支持匿名重新登录。

### 3.2 浏览商品

商品页同时请求两个接口：`/api/mobile/products`（商品列表：id/name/price/imageUrl）和 `/api/devices/D01-F1/stock`（库存数量），合并展示每件商品的图片、价格和实时库存。

商品类别与硬件端 YOLO 识别结果对应：汽水（soda）、农夫山泉（nongfu）、润田（runtian）、酸奶（yogurt），共 4 类。

### 3.3 开门购物

用户在商品页点击"开门购买"或选择具体商品，触发开门流程：

1. 弹出确认框（若已选商品则显示商品名和价格）
2. 调用 `POST /api/mobile/open`，发送 `userId` + `deviceId`（固定 "D01-F1"）
3. 服务器通过 MQTT 向设备下发 `unlock` 指令
4. 设备舵机开锁 → 用户开门取货 → 关门 → 设备拍照 + YOLO 识别 + 上报
5. 服务器根据识别结果生成订单，返回 `batchId`
6. 小程序保存 `batchId` 到本地存储

开门成功后，小程序开始**轮询订单**：每 2 秒调用 `/api/mobile/order/{batchId}`，检测订单是否已生成（`found: true`）。30 秒超时后提示用户重试。订单生成后弹出模态框展示商品明细和总金额。

### 3.4 查看订单

首页展示最近 5 条订单；订单 Tab 展示全部历史订单，支持下拉刷新和上拉加载更多。每条订单显示：订单号（batchId 前 8 位）、商品摘要（第一件商品名 + 件数）、总金额、支付状态（待支付/已支付）。

### 3.5 支付结算

订单详情页提供支付按钮。调用 `POST /api/mobile/pay`，发送 `orderId` + `userId`。支付成功后刷新订单状态并标记 `refreshOrders` 标志，返回订单列表时自动重新加载。

### 3.6 未支付提醒

首页每次显示时检查是否存在未支付订单（`payStatus === 'UNPAID'`），如有则弹窗提醒，用户可点击直接跳转到该订单详情页完成支付。

## 4. API 接口

所有请求通过 `app.request()`（管理端，含 Basic Auth）或 `app.mobileRequest()`（用户端，无鉴权）封装，自动处理 loading 状态和 401 错误。

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/mobile/products` | GET | 获取商品列表 |
| `/api/mobile/open` | POST | 开门请求，body: {userId, deviceId} |
| `/api/mobile/order/{batchId}` | GET | 查询订单（含商品明细、金额、支付状态） |
| `/api/mobile/orders?userId=` | GET | 用户历史订单列表 |
| `/api/mobile/pay` | POST | 支付，body: {orderId, userId} |
| `/api/devices/D01-F1/stock` | GET | 设备库存查询 |

---

## 5. 用户状态管理

- **登录状态**：`userId`（UUID）+ `userInfo`（微信昵称/头像）+ `token` 存储于本地
- **匿名用户**：首次启动自动生成 UUID 作为 `userId`，登录后绑定微信信息
- **订单状态**：UNPAID（待支付）、PAID（已支付）、FAILED（失败）
- **网络监听**：商品页注册 `onNetworkStatusChange`，断网恢复后自动重新加载数据

---

## 6. 用户体验设计

- 所有页面均检查登录状态，未登录自动引导至登录页
- 网络请求默认显示 loading 动画，失败提供重试入口
- 空数据状态使用 `empty-state` 组件，避免空白页
- 商品图片本地打包（`1.jpg` ~ `4.png`），减少网络依赖
- 开门后等待订单采用轮询 + 超时机制，避免用户无限等待
- 首页每次 `onShow` 自动刷新订单和未支付检查
- 支持分享到微信聊天和朋友圈
